# `_shared/` — canonical Python helpers vendored into skills

`_shared/` holds Python helpers that are **genuinely shared** across more than one skill.
It is a top-level repo dir, deliberately **outside** the `skills/` embed root, so the `yf`
binary never treats it as a skill (skill enumeration requires a `SKILL.md`). It ships **zero**
`yf` Rust changes: skills are installed as a verbatim embed→deploy of `skills/`, and the
vendored copies live *inside* each consuming script — install copies them like any other line.

## What lives here

| File | Role |
|:-----|:-----|
| `active_set.py` | Canonical active-set classifier (`classify_active` + `Edge`, `ActiveSetReport`, the `_directly_active`/`_has_owner`/`_is_closed` helpers, and the `ACTIVE_*`/`PARENT_CHILD`/`OPEN`/`IN_PROGRESS`/`CLOSED_STATUSES`/`GATE_TYPE` constants). The single source of truth. |
| `sync.py` | Repo-time sync tool — regenerates the vendored region in each consumer from canonical; `--check` reports divergence. |
| `README.md` | This file. |

## The pattern: regenerate-the-fenced-region (vendoring shape (b))

Skills must be **independently installable** — they cannot `import` each other or a sibling
package. So a genuinely shared helper is **vendored as a committed copy** inside each consuming
script, not imported. To avoid the silent rot of N hand-maintained copies (issue #15), the copy
is **generated**, not hand-edited:

1. The canonical body lives between the `# >>> BEGIN active-set classifier (canonical) >>>` and
   `# <<< END active-set classifier (canonical) <<<` markers in `active_set.py`.
2. Each consuming script carries a matching **marker-fenced region**
   (`# >>> BEGIN active-set classifier (generated by _shared/sync.py — do not edit) >>>` …
   `# <<< END active-set classifier <<<`).
3. `uv run _shared/sync.py` overwrites each consumer's region body in-place from canonical.
   It introduces **no** sibling `import` and **no** new file — each script stays self-contained,
   so the single-file `spec_from_file_location` test loaders and skill README layout fences are
   untouched.

Consumers (the canonical→copy map, declared in `sync.py`):

- `skills/yf-beads-hygiene/scripts/beads_hygiene.py`
- `skills/yf-beads-upstream/scripts/upstream.py`

## Authority and the enforcement point

`active_set.py` is the single **fixed-authority** source; each vendored region is **derived**.
Two enforcement surfaces keep the copies honest:

- **`yf-drift-check` on-edit trigger (primary).** DRIFT-CHECK.md declares
  `e-active-set-copy-hygiene` and `e-active-set-copy-upstream` as `value-equal` edges from the
  canonical to each region. Editing the canonical **or** either copy fires the check; a divergent
  region FAILs the **copy** (never the canonical).
- **`_shared/sync.py --check` (CI / manual backstop).** Exits non-zero on any region divergence.

Because the copies are *generated*, the normal rot vector — hand-editing one copy and forgetting
the other — is removed: edit canonical, re-run `sync.py`, commit the regenerated scripts.
