## Project environment

`beads-skills` (the working copy of the published `dixson3/yoshiko-flow` repo) is a collection of
**beads-backed Claude Code skills** (the `yf-*` family: yf-plan, yf-research, yf-drift-check,
yf-beads-*, yf-markdown-lint, etc.) plus a Rust **`yf` CLI** that installs/upgrades/verifies the
skills (embeds `skills/` verbatim via `rust-embed`) and runs `yf preflight`/`yf doctor`. Skills live
under `skills/<name>/` (each with `SKILL.md`, `README.md`, optional `spec/`, `scripts/` PEP-723
Python helpers run via `uv run`, `agents/`, `protocols/` always-loaded rules aggregated into
`~/.claude/rules/YOSHIKO_FLOW.md` by the `yf` flow machinery). Shared Python helpers live in the
top-level `_shared/` package (plan-014), vendored into consuming scripts as marker-fenced regions
regenerated by `_shared/sync.py`. Task tracking is **beads (`bd`)** over a Dolt DB in `.beads/`
(gitignored). Per-repo content-agreement is policed by an approved `DRIFT-CHECK.md` manifest. This
plan adds a new sibling skill, `yf-change-validation`. Non-obvious: there is **no `install.sh`** —
install is the `yf` binary; rules are data-driven from `skills/*/protocols/`.
